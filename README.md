Проанализированы существующие программные аналоги (ИРБИС, Koha), специфика учета библиотечных фондов и ключевые бизнес-процессы выдачи и возврата литературы. ## Введение <a id="введение"></a>
Актуальность темы обусловлена высокой нагрузкой на сотрудников современных библиотечных учреждений и необходимостью оперативной обработки информации о поступающем книжном фонде и зарегистрированных читателях. Согласно статистическим данным, ежегодно библиотеки обслуживают миллионы читателей, при этом значительная доля первичного документооборота и учета формуляров по-прежнему ведется в бумажном или неавтоматизированном виде [[8]](#source8), [[10]](#source10).

Современные исследования показывают, что внедрение информационных систем автоматизации библиотечной деятельности позволяет:
* сократить время обслуживания одного читателя на 40-55%;
* снизить количество ошибок при оформлении выдачи и возврата документов на 60-70%;
* повысить пропускную способность абонемента библиотеки на 25-30% [[10]](#source10).

**Цель работы:** проектирование архитектуры информационной системы библиотеки, предназначенной для автоматизации процессов ведения электронного каталога, учета читателей, регистрации транзакций выдачи и возврата литературы с возможностью контроля сроков.

## 1. Анализ предметной области <a id="1-анализ-предметной-области"></a>

Управление библиотечными фондами представляет собой комплексную деятельность по комплектованию, учету, хранению, выдаче книг. Основные бизнес-процессы представлены в таблице 1.1:

| Процесс | Входные данные | Результат |
|---|---|---|
| **Регистрация читателя** | Паспортные данные, ФИО, контактная информация | Формирование электронной карточки читателя, генерация и присвоение уникального идентификатора в базе данных. |
| **Оформление выдачи** | Уникальный ID читателя, отсканированный ISBN книги | Создание связанной транзакции (LoanRecord), фиксация системного времени выдачи, блокировка экземпляра для других читателей. |
| **Контроль сроков** | Плановая дата возврата, текущее системное время | Фоновый пересчет количества дней просрочки для активных транзакций, расчет суммы задолженности. |

## 3. Проектирование архитектуры <a id="3-проектирование-архитектуры"></a>

Для реализации прототипа ИС АРБ выбран стек технологий, ориентированный на производительность и портативность:
* **Язык программирования:** C++ (стандарт C++14/17). Использование компиляторов GCC и MSVC обеспечивает кроссплатформенную компиляцию кода для Windows и Linux.
* **Библиотека:** STL (Standard Template Library). Для быстрого поиска книг применен контейнер `std::unordered_map` (хеш-таблица), обеспечивающий константную сложность поиска O(1).
* **СУБД:** Встраиваемая база данных SQLite, что исключает необходимость развертывания выделенного сервера баз данных.

Спроектированный архитектурный каркас ИС АРБ готов к программной реализации. Архитектура обладает достаточной гибкостью для дальнейшего масштабирования и внедрения в работу малых учебных и муниципальных библиотек.
