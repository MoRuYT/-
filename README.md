## Аннотация[span_5](start_span)[span_5](end_span)
Проанализированы существующие программные аналоги (ИРБИС, Koha), специфика учета библиотечных фондов и ключевые бизнес-процессы выдачи и возврата литературы[span_6](start_span)[span_6](end_span). Спроектирована многоуровневая архитектура программного приложения на языке C++ с использованием СУБД SQLite[span_7](start_span)[span_7](end_span). Выбраны и научно обоснованы шесть паттернов проектирования (Facade, Repository, Strategy, State, Observer, Factory Method), а также проверено соблюдение принципов SOLID[span_8](start_span)[span_8](end_span).

**Ключевые слова:** информационная система, автоматизация библиотеки, архитектура программного обеспечения, C++, паттерны проектирования, SOLID[span_9](start_span)[span_9](end_span).

## Введение <a id="введение"></a>[span_10](start_span)[span_10](end_span)
Актуальность темы обусловлена высокой нагрузкой на сотрудников современных библиотечных учреждений и необходимостью оперативной обработки информации о поступающем книжном фонде и зарегистрированных читателях[span_11](start_span)[span_11](end_span). Согласно статистическим данным, ежегодно библиотеки обслуживают миллионы читателей, при этом значительная доля первичного документооборота и учета формуляров по-прежнему ведется в бумажном или неавтоматизированном виде [[8]](#source8), [[10]](#source10)[span_12](start_span)[span_12](end_span).

Современные исследования показывают, что внедрение информационных систем автоматизации библиотечной деятельности позволяет:[span_13](start_span)[span_13](end_span)
* сократить время обслуживания одного читателя на 40-55%[span_14](start_span)[span_14](end_span);
* снизить количество ошибок при оформлении выдачи и возврата документов на 60-70%[span_15](start_span)[span_15](end_span);
* повысить пропускную способность абонемента библиотеки на 25-30% [[10]](#source10)[span_16](start_span)[span_16](end_span).

**Цель работы:** проектирование архитектуры информационной системы библиотеки, предназначенной для автоматизации процессов ведения электронного каталога, учета читателей, регистрации транзакций выдачи и возврата литературы с возможностью контроля сроков[span_17](start_span)[span_17](end_span).

## 1. Анализ предметной области <a id="1-анализ-предметной-области"></a>[span_18](start_span)[span_18](end_span)

Управление библиотечными фондами представляет собой комплексную деятельность по комплектованию, учету, хранению, выдаче книг[span_19](start_span)[span_19](end_span). Основные бизнес-процессы представлены в таблице 1.1:[span_20](start_span)[span_20](end_span)

| Процесс | Входные данные | Результат |
|---|---|---|
| **Регистрация читателя** | Паспортные данные, ФИО, контактная информация[span_21](start_span)[span_21](end_span) | Формирование электронной карточки читателя, генерация и присвоение уникального идентификатора в базе данных[span_22](start_span)[span_22](end_span). |
| **Оформление выдачи** | Уникальный ID читателя, отсканированный ISBN книги[span_23](start_span)[span_23](end_span) | Создание связанной транзакции (LoanRecord), фиксация системного времени выдачи, блокировка экземпляра для других читателей[span_24](start_span)[span_24](end_span). |
| **Контроль сроков** | Плановая дата возврата, текущее системное время[span_25](start_span)[span_25](end_span) | Фоновый пересчет количества дней просрочки для активных транзакций, расчет суммы задолженности[span_26](start_span)[span_26](end_span). |

## 3. Проектирование архитектуры <a id="3-проектирование-архитектуры"></a>[span_27](start_span)[span_27](end_span)

Для реализации прототипа ИС АРБ выбран стек технологий, ориентированный на производительность и портативность:[span_28](start_span)[span_28](end_span)
* **Язык программирования:** C++ (стандарт C++14/17)[span_29](start_span)[span_29](end_span). Использование компиляторов GCC и MSVC обеспечивает кроссплатформенную компиляцию кода для Windows и Linux[span_30](start_span)[span_30](end_span).
* **Библиотека:** STL (Standard Template Library)[span_31](start_span)[span_31](end_span). Для быстрого поиска книг применен контейнер `std::unordered_map` (хеш-таблица), обеспечивающий константную сложность поиска O(1)[span_32](start_span)[span_32](end_span).
* **СУБД:** Встраиваемая база данных SQLite, что исключает необходимость развертывания выделенного сервера баз данных[span_33](start_span)[span_33](end_span).

## Заключение <a id="заключение"></a>[span_37](start_span)[span_37](end_span)
Спроектированный архитектурный каркас ИС АРБ готов к программной реализации[span_38](start_span)[span_38](end_span). Архитектура обладает достаточной гибкостью для дальнейшего масштабирования и внедрения в работу малых учебных и муниципальных библиотек[span_39](start_span)[span_39](end_span).
